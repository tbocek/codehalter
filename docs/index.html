For coding: an IDE, not a chat window

A holder for your code.

codehalter is an ACP coding agent that connects Zed, VS Code, or JetBrains to any OpenAI-compatible model, built for and focused on local LLMs. It has no UI of its own: your editor is the front end, not a CLI. Plan → execute → verify → document, all through the ACP bridge.

Designed to be held, not trusted blindly.

Every primitive assumes you want to stay in the loop.

Devcontainer-first

Scaffolds a throwaway devcontainer, Alpine, Arch, Debian, Fedora, or Ubuntu, and runs entirely inside it, so edits and build/test commands stay off your host. It works with your real .git and identity, but is instructed never to commit or push on its own, that stays your call.

Reviewable diffs

Edits land as diffs you approve in Zed. Interactive mode gates every file write and command; Autopilot runs the batch hands-free when you trust it.

Self-verify, then replan

Each subtask carries its own verify recipe and runs it before declaring done. A failure feeds a fresh plan with the context, up to 20 attempts, bailing early when the model keeps rephrasing the same problem.

Prefix-cache discipline

The rendered prompt stays byte- and position-stable turn over turn, so your server's KV cache stays ~90%+ warm. The only deliberate break is a new MCP server coming online.

Code intelligence, every language

Real go-to-definition, find-references and symbol search, not just for Go. gopls is wired in out of the box; TypeScript, C/C++ (clangd), Rust, Python, … bridge their LSP to MCP via lsmcp. Bring any other MCP server via mcp.toml. (non-Go bridges experimental)

Smart background compaction

Every turn, a separate LLM slot condenses that turn into structured notes in the background while you read the reply, so when the context nears the model's n_ctx, compaction is already prepared and fast. Needs ≥2 parallel slots.

Commit messages on the fly

A background drafter refreshes a commit message matching your working tree after every turn, so the moment you ask to commit, it's already written. The agent never commits on its own; when you ask, it runs the commit with the ready message (or hands you the exact command).

Slash-command macros

Type /grill-me and the agent interviews you about a plan until the design is resolved. Any /<name> expands a TEMPLATE-<name>.md into a full prompt, drop your own for reusable team workflows.

Sets up new projects

Point it at a fresh repo and it onboards the whole thing: scaffolds the devcontainer, detects the stack, seeds skills, and offers one-click fix cards to install what's missing, gopls/node/clang, an LSP bridge, a broken mcp.toml, doing it for you when you accept.
